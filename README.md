# 🔐 解码风云

[![GitHub Pages](https://img.shields.io/badge/链接-GitHub%20Pages-blue)](https://Bi-an-lai-ke.github.io/decode-the-flux/)
[![MIT License](https://img.shields.io/badge/license-MIT-green)](LICENSE)

10种素材中可重复地选择两种，可以按一定规则无顺序地组成子码。10个不同的子码按照一定顺序组成特定密码。请利用强大的推理力、观察力、记忆力、创造力，从100个候选子码中识别出合规组成子码，并利用尽可能少的尝试次数，尽快推断出正确密码。

**四种反馈**：  
  - ✅ 完全正确：子码与位置均正确，此后该栏位将会被锁定  
  - 🔄 栏位错误：子码存在于密码中，但位置不对  
  - 🔶 部分正确：仅一个素材匹配  
  - ❌ 完全错误：子码及素材均不参与密码

**三种模式**：  
  - **练习模式**：手动放置素材合成合规子码，可查看/编辑答案，适合学习规则、记忆子码组成、优化算法。  
  - **计时模式**：从100个子码方阵中直接选择子码进行尝试，此模式中存在干扰项子码。  
  - **比赛模式**：需要按照计时模式的规则连续破解3组密码。  

目前已有稳定7次尝试得正解的算法，6次大概率可行。根据信息论，理论上可以做到第5次提交即为正确尝试，但这个算法应当如何实现？欢迎各位共同探讨。
